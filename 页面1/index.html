5.连接Xshellx,配置免密操作

Xshell 支持 SSH 协议，这是一种加密的网络协议，用于在不安全的网络中安全地远程登录另一台计算机。这意味着通过 Xshell 连接至你的 Hadoop 环境可以保护数据传输的安全性，防止信息泄露。还能方便地管理多个终端，如果要在多个终端输入相同的命令，可以只用在某个终端里输入一次。

通过 Xshell 远程连接到 Linux 系统，以便安装必要的软件包（如 Java）、配置环境变量

命令解释：

  • cat: 这个命令用于连接文件并打印到标准输出设备（通常是终端屏幕）。在这里，它被用来显示文件的内容。

  • id_rsa1.pub: 这是slave1的公钥文件，包含了用于认证的一个公开密钥。

  • >>: 这是一个重定向符号，表示将前一个命令的标准输出追加到后面的文件中。与单个大于号 > 不同的是，>> 不会覆盖目标文件内容，而是将其添加到文件末尾。

  • authorized_keys: 这是 SSH 配置中的一个重要文件，位于 .ssh 目录内。该文件包含了所有允许通过公钥认证登录到该用户的公共密钥。

  • scp: Secure Copy 的缩写，这是一个基于 SSH 协议的文件传输命令，用于在本地主机和远程主机之间安全地复制文件。

  • /root/.ssh/authorized_keys: 这是源文件路径，表示从master的 /root/.ssh/ 目录中复制 authorized_keys 文件。

  • root@slave1: 这指定了目标主机和用户信息。

    • root: 表示你要以 root 用户的身份登录到目标主机。
    • slave1: 这是目标主机的名字或 IP 地址。

  • :/root/.ssh: 这是目标主机上的目录路径，表示将文件复制到目标主机的 /root/.ssh/ 目录下。

作用: 将本地机器上的 /root/.ssh/authorized_keys 文件复制到名为 slave1 的远程主机的 /root/.ssh/ 目录下。这一步是为了确保 slave1 主机能够识别并信任来自其他节点（如 master 和 slave2）的公钥，从而实现无密码 SSH 登录。

作用: 此命令的作用是将 id_rsa1.pub 文件中的内容追加到 authorized_keys 文件末尾。这意味着任何拥有与 id_rsa1.pub 中公钥对应的私钥的用户都可以无密码登录到此服务器。

将master的 /root/.ssh/authorized_keys 文件复制到名为 slave1 和slave2的远程主机的 /root/.ssh/ 目录下。这一步是为了确保 slave1 和slave2能够识别并信任来自其他节点（如 master 和 slave2）的公钥，从而实现无密码 SSH 登录。
